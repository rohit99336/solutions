यह पैकेज Laravel में **Security (सुरक्षा)** के लिए बहुत महत्वपूर्ण है। सरल शब्दों में कहें तो `mews/purifier` आपके द्वारा प्राप्त किए गए HTML इनपुट को **"साफ़" (Clean/Sanitize)** करने का काम करता है।

इसका मुख्य काम **XSS (Cross-Site Scripting)** हमलों को रोकना है।

---

### Install:

```
composer require mews/purifier
```

### Use:
```
{!! clean($email->body_html) !!}
```

### **यह क्या करता है?**

जब आप किसी यूजर से डेटा लेते हैं (जैसे कि किसी **WYSIWYG Editor** जैसे CKEditor या TinyMCE के माध्यम से), तो यूजर उसमें खतरनाक JavaScript कोड भेज सकता है।

**उदाहरण के लिए:**
अगर कोई यूजर कमेंट बॉक्स में यह लिख दे:
`<script>alert('Hacked!');</script> <p>Hello!</p>`

* **बिना Purifier के:** अगर आप इसे सीधा डिस्प्ले करेंगे, तो ब्राउज़र में JavaScript चल जाएगी (XSS Attack)।
* **Purifier के साथ:** यह खतरनाक `<script>` टैग को हटा देगा और सिर्फ सुरक्षित `<p>Hello!</p>` को रखेगा।

---

### **इसका उपयोग कब करें?**

1. **Admin Panel में:** जब आप ईमेल बॉडी, ब्लॉग पोस्ट या कोई भी रिच टेक्स्ट सेव कर रहे हों।
2. **User Input:** जहाँ आप यूजर को कुछ HTML टैग्स (जैसे `<b>`, `<i>`, `<ul>`) इस्तेमाल करने की अनुमति देना चाहते हैं, लेकिन सुरक्षा भी बनाए रखना चाहते हैं।

---

### **कोड में कैसे इस्तेमाल करें?**

पैकेज इंस्टॉल करने के बाद, आप इसे अपने Controller में ऐसे उपयोग कर सकते हैं:

```php
use Mews\Purifier\Facades\Purifier;

public function store(Request $request)
{
    // यह इनपुट से सभी खतरनाक स्क्रिप्ट्स को निकाल देगा
    $clean_html = Purifier::clean($request->input('email_body'));

    AdminMail::create([
        'body' => $clean_html,
        // अन्य फील्ड्स...
    ]);
}

```

---

### **प्रमुख विशेषताएं:**

* **White-listing:** आप कॉन्फ़िगर कर सकते हैं कि कौन से HTML टैग्स (जैसे `<a>`, `<img>`) "Allowed" हैं और कौन से नहीं।
* **HTML5 Support:** यह आधुनिक HTML5 टैग्स को अच्छी तरह समझता है।
* **Auto-Correction:** अगर यूजर कोई टैग खोलकर बंद करना भूल गया है (जैसे `<div>hello`), तो यह उसे अपने आप ठीक कर देता है (`<div>hello</div>`)।

---

### **क्या आपको इसकी ज़रूरत है?**

चूँकि आप एक **Email Fetching** सिस्टम बना रहे हैं, तो यह पैकेज आपके लिए **बहुत ज़रूरी** है। ईमेल में अक्सर बहुत सारे HTML टैग्स और बाहरी स्क्रिप्ट्स होती हैं। उन्हें अपने डेटाबेस में सेव करने और अपनी वेबसाइट पर दिखाने से पहले `Purifier` से साफ़ करना सुरक्षा के लिहाज से सबसे अच्छा तरीका (Best Practice) है।

क्या आप चाहते हैं कि मैं इसकी `config` फाइल को कस्टमाइज़ करने का तरीका बताऊं ताकि आप अपनी ज़रूरत के हिसाब से टैग्स को Allow या Block कर सकें?

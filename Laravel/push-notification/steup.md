 **Laravel 12** प्रोजेक्ट में पुश नोटिफिकेशन (Push Notifications) जोड़ सकते हैं। इसे करने के लिए **Web Push** तकनीक का इस्तेमाल किया जाता है, जिससे यूज़र को तब भी नोटिफिकेशन मिलता है जब आपकी वेबसाइट उनके ब्राउज़र में खुली न हो।

Laravel 12 में इसे लागू करने के सबसे बेहतरीन और आधुनिक तरीके यहाँ दिए गए हैं:

---

### 1. मुख्य तरीके (Two Best Ways)

| तरीका | माध्यम (Channel) | क्यों चुनें? |
| --- | --- | --- |
| **Pusher + Beams** | थर्ड-पार्टी सर्विस | यह सेटअप करने में बहुत आसान है और इसमें नोटिफिकेशन डिलीवरी की गारंटी होती है। |
| **Firebase (FCM)** | Google की सर्विस | यह पूरी तरह फ्री है और Android/iOS/Web तीनों पर बहुत अच्छा काम करता है। |

---

### 2. इसे कैसे लागू करें? (R&D आधारित स्टेप्स)

चूंकि आप **Laravel 12** इस्तेमाल कर रहे हैं, तो हम Laravel के इन-बिल्ट **Notification System** का उपयोग करेंगे।

#### स्टेप 1: नोटिफिकेशन ड्राइवर इंस्टॉल करें

अगर आप Firebase इस्तेमाल करना चाहते हैं, तो आपको एक कम्युनिटी पैकेज की जरूरत होगी (जैसे `laravel-notification-channels/fcm`):

```bash
composer require laravel-notification-channels/fcm

```

#### स्टेप 2: सर्विस वर्कर (Service Worker) बनाना

पुश नोटिफिकेशन तभी काम करते हैं जब ब्राउज़र बंद हो, इसके लिए `public` फोल्डर में एक `firebase-messaging-sw.js` फाइल बनानी होती है। यह फाइल बैकग्राउंड में चलती रहती है और सर्वर से आने वाले सिग्नल को पकड़ती है।

#### स्टेप 3: डेटाबेस में टोकन स्टोर करना

हर यूज़र का एक यूनिक "Device Token" होता है। जब यूज़र नोटिफिकेशन की अनुमति (Permission) देता है, तो आपको उस टोकन को अपने `users` टेबल में स्टोर करना होगा ताकि आप टारगेटेड मैसेज भेज सकें।

---

### 3. Laravel 12 कोड उदाहरण

मान लीजिए आप किसी कस्टमर को ऑर्डर का नोटिफिकेशन भेजना चाहते हैं, तो आपका नोटिफिकेशन क्लास कुछ ऐसा दिखेगा:

```php
namespace App\Notifications;

use Illuminate\Notifications\Notification;
use NotificationChannels\Fcm\FcmChannel;
use NotificationChannels\Fcm\FcmMessage;

class OrderNotification extends Notification
{
    public function via($notifiable)
    {
        // यहाँ आप तय करते हैं कि नोटिफिकेशन कहाँ जाएगा
        return [FcmChannel::class, 'database'];
    }

    public function toFcm($notifiable)
    {
        return (new FcmMessage(title: 'नया संदेश!', body: 'आपका ऑर्डर पैक हो गया है।'))
            ->setData(['order_id' => '123']);
    }
}

```

---

### 4. यूज़र प्रोजेक्ट बंद रखे तो क्या होगा?

यही पुश नोटिफिकेशन की खासियत है!

* **डेस्कटॉप पर:** यूज़र अगर क्रोम (Chrome) या कोई भी ब्राउज़र इस्तेमाल कर रहा है, तो सिस्टम के कोने में (Windows/Mac की तरह) नोटिफिकेशन पॉप-अप होगा।
* **मोबाइल पर:** अगर उन्होंने आपकी वेबसाइट को "Add to Home Screen" (PWA) किया है, तो यह बिल्कुल एक असली मोबाइल ऐप की तरह नोटिफिकेशन भेजेगा।

### 5. क्या सावधानी रखें?

* **HTTPS:** पुश नोटिफिकेशन केवल सुरक्षित (HTTPS) साइट्स पर ही काम करते हैं। लोकलहोस्ट (localhost) पर यह टेस्टिंग के लिए चल जाएगा।
* **User Permission:** यूज़र को पहले "Allow Notifications" पर क्लिक करना होगा, तभी आप उन्हें मैसेज भेज पाएंगे।

**मेरी सलाह:** अगर आप पहली बार पुश नोटिफिकेशन लगा रहे हैं, तो **Firebase (FCM)** सबसे अच्छा और सस्ता (फ्री) विकल्प है।

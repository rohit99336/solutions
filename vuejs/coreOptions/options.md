`el`, `data`, और `methods` के अलावा, Vue.js इंस्टेंस या कॉम्पोनेन्ट में कई अन्य **मुख्य विकल्प (Core Options)** होते हैं जो कॉम्पोनेन्ट के व्यवहार (behavior), डेटा और लाइफसाइकिल (lifecycle) को नियंत्रित करते हैं।

यहाँ कुछ और **महत्वपूर्ण Core Options** दिए गए हैं जो Vue में अक्सर उपयोग होते हैं:

-----

## ⚙️ अन्य महत्वपूर्ण Vue Core Options

### 1\. `template` (टेम्प्लेट)

यह कॉम्पोनेन्ट के HTML स्ट्रक्चर को परिभाषित करता है।

| विशेषता | विवरण |
| :--- | :--- |
| **क्या है?** | एक स्ट्रिंग जो कॉम्पोनेन्ट का **मार्कअप (markup)** (HTML) प्रदान करती है। |
| **उपयोग** | यदि आप `.vue` फ़ाइलों का उपयोग नहीं कर रहे हैं और सीधे JavaScript में कॉम्पोनेन्ट को परिभाषित कर रहे हैं। यदि `template` विकल्प मौजूद है, तो यह **`el`** द्वारा निर्धारित HTML को ओवरराइड कर देता है। |

#### उदाहरण:

```javascript
new Vue({
  el: '#app',
  data: {
    title: 'My Custom App'
  },
  template: '<h1>{{ title }}</h1>' // यह #app के अंदर का HTML बदल देगा
});
```

-----

### 2\. `computed` (कंप्यूटेड प्रॉपर्टीज़)

यह डेटा पर आधारित जटिल गणनाओं (complex calculations) को हैंडल करने के लिए उपयोग होता है।

| विशेषता | विवरण |
| :--- | :--- |
| **क्या है?** | एक ऑब्जेक्ट जिसमें **फ़ंक्शन** होते हैं जो डेटा रिटर्न करते हैं। |
| **उपयोग** | ये फ़ंक्शन `data` की प्रॉपर्टीज़ पर निर्भर करते हैं। जब भी निर्भर डेटा बदलता है, Vue इन कंप्यूटेड प्रॉपर्टीज़ को **स्वचालित रूप से कैशे (automatically caches)** और अपडेट करता है। ये `data` की तरह ही टेम्प्लेट में उपयोग होते हैं। |

#### उदाहरण:

यदि आपके पास `firstName` और `lastName` है, तो `fullName` एक कंप्यूटेड प्रॉपर्टी होगी:

```javascript
new Vue({
  data: {
    firstName: 'Ravi',
    lastName: 'Kumar'
  },
  computed: {
    // यह firstName और lastName पर निर्भर करता है
    fullName() {
      return `${this.firstName} ${this.lastName}`;
    }
  }
});
```

> **टेम्प्लेट में उपयोग:** `{{ fullName }}`

-----

### 3\. `watch` (वॉचर्स)

यह किसी **रिएक्टिव डेटा प्रॉपर्टी** में बदलाव होने पर एक **साइड इफ़ेक्ट (side effect)** या एसिंक्रोनस ऑपरेशन (async operation) को निष्पादित करने के लिए उपयोग होता है।

| विशेषता | विवरण |
| :--- | :--- |
| **क्या है?** | एक ऑब्जेक्ट जिसमें उस डेटा प्रॉपर्टी का नाम होता है जिसे आप **देखना (watch)** चाहते हैं, और उसके बदलाव पर कॉल होने वाला एक फ़ंक्शन होता है। |
| **उपयोग** | डेटा बदलने पर किसी API को कॉल करना, समयबद्ध (debounced) ऑपरेशन करना, या जटिल लॉजिक निष्पादित करना। |

#### उदाहरण:

जब `question` डेटा बदलता है, तो एक फ़ंक्शन कॉल करें:

```javascript
new Vue({
  data: {
    question: '',
    answer: 'I am waiting for your question...'
  },
  watch: {
    // 'question' डेटा प्रॉपर्टी को देख रहा है
    question(newQuestion, oldQuestion) {
      if (newQuestion.includes('?')) {
        this.answer = 'Processing your question...';
        // यहाँ एक API कॉल जैसा जटिल लॉजिक आ सकता है
      }
    }
  }
});
```

-----

### 4\. `components` (घटक)

यह वह ऑब्जेक्ट है जहाँ आप वर्तमान कॉम्पोनेन्ट के अंदर उपयोग किए जाने वाले **अन्य Vue कॉम्पोनेन्ट्स** को रजिस्टर करते हैं।

| विशेषता | विवरण |
| :--- | :--- |
| **क्या है?** | एक ऑब्जेक्ट जहाँ `key` कॉम्पोनेन्ट का नाम होता है और `value` कॉम्पोनेन्ट की परिभाषा होती है। |
| **उपयोग** | Vue का मुख्य सिद्धांत है कि UI को छोटे, पुन: प्रयोज्य (reusable) कॉम्पोनेन्ट्स में विभाजित किया जाए। यह विकल्प उन कॉम्पोनेन्ट्स को उपलब्ध कराता है। |

#### उदाहरण:

मान लीजिए आपने एक `MyButton` कॉम्पोनेन्ट बनाया है:

```javascript
// MyButton कॉम्पोनेन्ट की परिभाषा
const MyButton = { 
    template: '<button><slot></slot></button>' 
};

// पैरेंट कॉम्पोनेन्ट
new Vue({
  el: '#app',
  components: {
    // MyButton को रजिस्टर किया
    'my-button': MyButton 
  },
  template: '<div id="app"><my-button>Click Me</my-button></div>'
});
```

> **टेम्प्लेट में उपयोग:** `<my-button>...</my-button>`

-----

### 5\. `props` (प्रॉप्स)

यह कॉम्पोनेन्ट को **पैरेंट कॉम्पोनेन्ट** से डेटा पास करने के लिए उपयोग किया जाता है।

| विशेषता | विवरण |
| :--- | :--- |
| **क्या है?** | एक **ऐरे** या एक **ऑब्जेक्ट** जो उन गुणों (properties) को घोषित करता है जिन्हें यह कॉम्पोनेन्ट बाहर से स्वीकार करता है। |
| **उपयोग** | कॉम्पोनेन्ट्स को **पुन: प्रयोज्य** बनाने के लिए। डेटा प्रॉप्स के माध्यम से नीचे की ओर (पैरेंट से चाइल्ड तक) प्रवाहित होता है। |

#### उदाहरण:

```javascript
// एक चाइल्ड कॉम्पोनेन्ट
const Greeting = {
  // 'userName' prop को स्वीकार करता है
  props: ['userName'], 
  template: '<h2>Hello, {{ userName }}!</h2>'
};

// पैरेंट कॉम्पोनेन्ट
new Vue({
  el: '#app',
  components: { 'app-greeting': Greeting },
  data: { user: 'Priya' },
  template: '<app-greeting :user-name="user"></app-greeting>' // डेटा पास करना
});
```

## ⏳ लाइफ़साइकिल हुक्स (Lifecycle Hooks)

ये विशेष `methods` होते हैं जो कॉम्पोनेन्ट के जीवनकाल (creation, mounting, updating, destruction) के अलग-अलग चरणों में अपने आप कॉल होते हैं।

सबसे अधिक उपयोग किए जाने वाले हुक्स में शामिल हैं:

1.  **`created()`:** कॉम्पोनेन्ट बनने के बाद (डेटा उपलब्ध है, लेकिन DOM अभी तक माउंट नहीं हुआ है)।
2.  **`mounted()`:** कॉम्पोनेन्ट DOM में इंजेक्ट (Mount) होने के बाद। (DOM मैनिपुलेशन यहाँ किया जा सकता है)।
3.  **`updated()`:** कॉम्पोनेन्ट का डेटा बदलने और DOM अपडेट होने के बाद।

#### उदाहरण:

```javascript
new Vue({
  data: {
    status: 'Initializing'
  },
  created() {
    // कॉम्पोनेन्ट बन गया, डेटा एक्सेस किया जा सकता है
    console.log('Component created. Data status:', this.status);
    this.status = 'Ready';
  },
  mounted() {
    // कॉम्पोनेन्ट HTML में है
    console.log('Component mounted to the DOM.');
  }
});
```

ये सभी विकल्प Vue कॉम्पोनेन्ट की परिभाषा का एक हिस्सा हैं, जो आपको शक्तिशाली और संरचित एप्लिकेशन बनाने में मदद करते हैं।

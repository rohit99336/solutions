Nginx को डिफ़ॉल्ट बनाना हैं, तो Apache को या तो बंद करना होगा या उसका पोर्ट (Port) बदलना होगा, क्योंकि डिफ़ॉल्ट रूप से दोनों एक ही रास्ते (**Port 80**) का इस्तेमाल करने की कोशिश करते हैं।

यहाँ समझिए कि आप यह कैसे कर सकते हैं:

---

### 1. Nginx को डिफ़ॉल्ट कैसे बनाएँ?

चूंकि Apache अभी Port 80 पर चल रहा है, इसलिए Nginx इंस्टॉल तो हो जाएगा लेकिन "Start" नहीं होगा। इसे सही करने के दो तरीके हैं:

* **तरीका A (Apache को बंद करना):** अगर Apache की ज़रूरत नहीं है, तो उसे पूरी तरह स्टॉप और डिसेबल कर दें।
```bash
sudo systemctl stop apache2
sudo systemctl disable apache2
sudo systemctl start nginx

```


* **तरीका B (पोर्ट बदलना):** अगर आप चाहते हैं कि दोनों सिस्टम में रहें, तो Apache का पोर्ट बदलकर `8080` कर दें और Nginx को `80` (Default) पर रहने दें।

### 2. क्या Nginx ऑटोमैटिक चलेगा?

हाँ, जैसे आपका Apache अभी अपने आप चलता है, वैसे ही Nginx इंस्टॉल होने के बाद एक "Service" बन जाता है। एक बार आप इसे `sudo systemctl enable nginx` कर देंगे, तो कंप्यूटर रीस्टार्ट होने पर यह अपने आप बैकग्राउंड में चालू हो जाएगा। बार-बार स्टार्ट करने की ज़रूरत नहीं पड़ेगी।

### 3. लारावेल (Laravel) के लिए Nginx क्यों बेहतर है?

लारावेल डेवलपर्स अक्सर Nginx पसंद करते हैं क्योंकि:

1. **Speed:** यह Apache के मुकाबले कम RAM लेता है और ज़्यादा तेज़ है।
2. **Configuration:** इसकी कॉन्फ़िगरेशन फाइल (`nginx.conf`) बहुत साफ और समझने में आसान होती है।
3. **Standard:** आज के समय में ज़्यादातर लाइव सर्वर (AWS, DigitalOcean) पर Nginx ही इस्तेमाल होता है।

---

### **लारावेल और Nginx का काम करने का तरीका**

जब आप लारावेल चलाते हैं, तो Nginx सीधे PHP को नहीं समझता। वह **PHP-FPM** नाम के एक टूल का इस्तेमाल करता है।

### * क्या करना चाहिए?**

अगर आप सिर्फ सीखने के लिए कर रहे हैं, तो मेरा सुझाव है:

1. Apache को स्टॉप कर दें।
2. Nginx इंस्टॉल करें।
3. Nginx को डिफ़ॉल्ट रहने दें।

**एक ज़रूरी चेतावनी:**
जब आप Nginx पर शिफ्ट होंगे, तो आपकी लारावेल की `.htaccess` फाइल काम **नहीं** करेगी। Nginx की 'Server Block' फाइल में लारावेल के रूल्स लिखने होंगे।

क्या आप चाहते हैं कि मैं **Nginx की वह कॉन्फ़िगरेशन फाइल** लिख कर दूँ जो लारावेल के लिए ज़रूरी है?
